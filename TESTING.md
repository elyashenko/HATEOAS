# Тестирование на продакшене

## Локальное тестирование с прод API

Для тестирования локального фронтенда с продакшн API:

1. Создайте файл `.env.local` (не коммитится в git):
   ```bash
   VITE_PROD_API_URL=https://hateoas.vercel.app/api
   ```

2. Запустите фронтенд с прод API:
   ```bash
   npm run dev:prod
   ```

   Или используйте стандартный dev с переменной окружения:
   ```bash
   VITE_PROD_API_URL=https://hateoas.vercel.app/api npm run dev:client
   ```

3. Откройте http://localhost:3001 и тестируйте функциональность

## Просмотр логов на Vercel

Логи Vercel можно посмотреть в:
- Vercel Dashboard → Project → Deployments → [выберите deployment] → Functions → [выберите функцию]
- Или через Vercel CLI: `vercel logs`

## Отладка проблем

Если что-то не работает:

1. Проверьте логи в консоли браузера (F12)
2. Проверьте логи на Vercel (см. выше)
3. Проверьте Network tab в DevTools - смотрите на запросы к `/api/posts/:id/archive`
4. Убедитесь, что запрос отправляется с правильным методом (POST) и URL

## Изменения для отладки

В коде добавлено детальное логирование для архивных запросов:
- В `api/[...path].ts` логируются детали запроса и ответа
- В `src/api/hateoas-client.ts` логируется выполнение действий

Логи видны в:
- Браузерная консоль (для клиентского кода)
- Vercel Functions logs (для серверного кода)
